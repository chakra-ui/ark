# Hover Card Angular Parity Audit

## Scope
- Angular files: `packages/angular/src/hover-card/*`, `packages/angular/src/hover-card/examples/*`, `packages/angular/src/hover-card/hover-card.spec.ts`
- React files: `packages/react/src/components/hover-card/*`, `packages/react/src/components/hover-card/examples/*`, `packages/react/src/components/hover-card/tests/*`
- Storybook/style files: `packages/angular/src/hover-card/hover-card.stories.ts`, `packages/angular/src/hover-card/hover-card-example-styles.ts`, `.storybook/modules/hover-card.module.css`

## Summary
- Status: Re-audited 2026-05-19. Angular hover-card is at parity with the React reference. Public API, story set, example structure, profile-card styling, and behavior tests all match the React surface. Presence flags (`lazyMount`, `unmountOnExit`) are intentionally Angular-specific via `<ark-presence>`; see Gap Matrix.
- Highest-risk gaps: none open. Previously flagged gaps (trigger `value` plumbing, missing `Context` story, profile-card layout/animations, spec coverage) are closed.
