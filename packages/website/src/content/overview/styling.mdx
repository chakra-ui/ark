Ark UI doesn't include built-in styling and is compatible with any styling
solution you prefer. In this guide, we'll use a simple `Tabs` component as an
example.

## Overview

### Styling parts

Every component part is assigned a `data-scope` and `data-part` attribute. The
`data-scope` represents the component name, the `data-part` represents the name
of the part.

> This was built to mirror the upcoming
> [CSS Scopes Specification](https://drafts.csswg.org/css-cascade-6/#example-463550a5)

### Styling states

Most interactive components have a `data-*` attribute that represents the
current state of the component.

For instance, the tab trigger and content element will have a `data-selected`
attribute when their `value` match the current active tab.

We extensively provide data attributes like `data-disabled`, `data-readonly`,
`data-focus`, `data-orientation` and so on, to help you style the component
based on its state.

## Using Panda CSS

Panda is a delightful, build-time CSS-in-JS framework that's designed to work
seamlessly with Ark UI. It's the recommended styling solution for Ark UI.

> Learn more about Panda CSS at [panda-css.com](https://panda-css.com).

### The styled factory

Panda CSS provides a `styled` function that enables you to style component parts
with ease. You can also leverage the `css` function to create simple styles.

```tsx
import { Tabs } from '@ark-ui/react'
import { styled } from 'styled-system/jsx' // panda generated directory

// using the styled function
const TabTrigger = styled(Tabs.Trigger, {
  base: {
    bg: 'gray.100',
    borderWidth: '1px',
    paddingTop: '3',
  },
  variants: {
    // ... define variants of your tab
  },
})

// using the css function
const listClassName = css({
  display: 'flex',
  flexWrap: 'wrap',
  fontSize: '14px',
  fontWeight: 'medium',
  textAlign: 'center',
  color: '#374151',
  borderBottomWidth: '1px',
  borderBottomColor: 'gray.200',
})

const Demo = () => (
  <Tabs.Root>
    <Tabs.List className={listClassName}>
      <TabTrigger value="react" />
    </Tabs.List>
    {/* … */}
  </Tabs.Root>
)
```

### Using slot recipes

Panda provides the idea of slot recipes, which are a collection of styles that
can be applied to different parts of a component. Slot recipes are defined using
the `cva` function provided by Panda.

```tsx
import { Tabs } from '@ark-ui/react'
import { sva } from 'styled-system/css' // panda generated directory

const tabStyle = sva({
  slots: ['list', 'trigger'],
  base: {
    // base styles for Tabs.List
    list: {},
    // base styles for Tabs.Trigger
    trigger: {},
  },
  variants: {
    // variant styles
  },
})

const Demo = () => {
  const styles = tabStyle()
  return (
    <Tabs.Root>
      <Tabs.List className={styles.list}>
        <Tab.Trigger className={styles.trigger} value="react" />
      </Tabs.List>
      {/* … */}
    </Tabs.Root>
  )
}
```

## Using Utility Classes

Utility classes offer a versatile means to style your components. This approach
proves particularly beneficial when using Ark UI in conjunction with a CSS
styling framework, such as Tailwind, UnoCSS or Bootstrap.

```tsx
<Tabs.Root>
  <Tabs.List className="flex flex-wrap text-sm font-medium text-center text-gray-500 border-b border-gray-200">
    <Tabs.Trigger
      value="react"
      className="inline-block p-4 text-blue-600 bg-gray-100 rounded-t-lg"
    >
      React
    </Tabs.Trigger>
  </Tabs.List>
</Tabs.Root>
```

## Using Inline Styles

Inline styles can also be used, although they're typically not recommended due
to potential conflicts and maintainability issues.

> Ark UI ensures that user-defined inline styles merge correctly with any
> built-in inline styles.

```tsx
<Tabs.Root>
  <Tabs.List
    style={{
      display: 'flex',
      flexWrap: 'wrap',
      fontSize: '14px',
      fontWeight: 'medium',
      textAlign: 'center',
      color: '#374151',
      borderBottom: '1px solid #e5e7eb',
    }}
  >
    <Tabs.Trigger
      value="react"
      style={{
        display: 'inline-block',
        padding: '16px',
        color: '#3b82f6',
        backgroundColor: '#f3f4f6',
        borderTopLeftRadius: '0.5rem',
        borderTopRightRadius: '0.5rem',
        borderBottom: '1px solid #e5e7eb',
      }}
    >
      React
    </Tabs.Trigger>
  </Tabs.List>
</Tabs.Root>
```

## Using CSS

As described earlier, you can use the provided `data-scope` and `data-part`
attribute to every component part to style them with ease.

Here's a simplified markup for the `Tabs` component:

```html
<div data-scope="tabs" data-part="root">
  <div role="tablist" data-scope="tabs" data-part="list">
    <button role="tab" data-scope="tabs" data-part="trigger" data-value="react">
      React
    </button>
  </div>
</div>
```

Here's an example of styling the `Tabs.List` and `Tabs.Trigger` parts using CSS

```css
[data-scope='tabs'][data-part='list'] {
  display: flex;
  flex-wrap: wrap;
  font-size: 14px;
  font-weight: medium;
  text-align: center;
  color: #374151;
  border-bottom: 1px solid #e5e7eb;
}
[data-scope='tabs'][data-part='trigger'] {
  display: inline-block;
  padding: 16px;
  color: #3b82f6;
  background-color: #f3f4f6;
  border-top-left-radius: 0.5rem;
  border-top-right-radius: 0.5rem;
  border-bottom: 1px solid #e5e7eb;
}
```

## Component Library

When using Ark UI with a component library, such as Chakra UI or Material UI,
you can use the `asChild` prop to swap the rendered element with your own while
retaining the original functionality.

> Note: If you're using React, it's important to ensure the element implements
> the `React.forwardRef` API to ensure the ref is forwarded correctly.

```tsx
import { Tabs } from '@ark-ui/react'
import { Button } from '@chakra-ui/react'

const Demo = () => (
  <Tabs.Root>
    <Tabs.List asChild>
      <List>
        <Tabs.Trigger value="react" asChild>
          <Button variant="outline">React</Button>
        </Tabs.Trigger>
      </List>
    </Tabs.List>
  </Tabs.Root>
)
```

## Conclusion

Styling in Ark UI is intentionally unopinionated to provide you with the maximum
flexibility you need to adapt the components to your own design system.

Whether you're using class names, inline styles, CSS selectors, or integrating
Ark UI with an existing UI library, the choice is yours.
